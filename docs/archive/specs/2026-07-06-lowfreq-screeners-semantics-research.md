# 低频筛选器精神含义与系统吸收关系研究结论

日期：2026-07-06

## 1. 文档目标

本文档用于固化当前 NeoTrade3 筛选器体系的研究结论，重点回答以下问题：

- 现有筛选器到底在试图识别什么类型的股票与机会。
- 当前 lowfreq engine 实际吸收了哪些筛选器精神，哪些尚未系统吸收。
- 后续在 `M1/M2/M3` 设计中，应如何保留这些团队经验资产，而不是机械搬运配置文件。

本文档是后续 `M1` 详细设计与 `M2/M3` 归位设计的参照材料，不直接触发实现。

## 2. 研究边界

本研究只基于仓库现状、筛选器配置、runtime 实现、页面/API 链路和 engine 代码证据，不做主观猜测。

本研究特别遵守以下原则：

- 不把筛选器名称当作语义本身。
- 不默认筛选器必须原封不动接入 engine。
- 先提炼“精神含义”，再讨论归属层级。

## 3. 研究对象

当前正式生效的筛选器主链共 7 个：

- `cup_handle_v4`
- `daily_hot_cold`
- `er_ban_hui_tiao`
- `jin_feng_huang`
- `shi_pan_xian`
- `yin_feng_huang`
- `zhang_ting_bei_liang_yin`

此外，还存在当前未接入正式 screener 主链、但与 engine 已有内置逻辑高度相关的对象：

- `lao_ya_tou_zhou_xian`

## 4. 当前系统事实

### 4.1 筛选器主链已存在完整运行体系

当前筛选器不是孤立配置文件，而是已有完整的：

- 注册表
- 页面入口
- API 调用链
- runtime 执行链
- ledger / artifact 产物链

说明：这些筛选器已经是系统正式资产，而不是实验性残留。

### 4.2 engine 当前只直接吸收了少数筛选器

目前 lowfreq engine 明确直接吸收的外部筛选器/相似逻辑主要有两类：

- `cup_handle_v4`
- 周线老鸭头相似逻辑（engine 内置实现）

其余筛选器的大部分经验语义，还没有以“独立可解释模块”的形式进入 lowfreq engine 主链。

### 4.3 当前已有“碎片化吸收”

虽然多数筛选器未被完整接线，但它们背后的一部分思想已经以碎片化形式进入 engine，例如：

- 结构确认
- soft retain
- 波段不确定降权
- focus gate
- 执行闸门
- tracking 观察

所以当前问题不是“完全没吸收”，而是“没有被系统化、层次化、可解释地吸收”。

## 5. 核心研究结论

### 5.1 筛选器本质上是团队经验沉淀出的模式假设

这些筛选器真正重要的，不是参数表本身，而是它们在试图捕捉的：

- 市场结构
- 资金行为
- 筹码与趋势形态
- 周期阶段位置
- 噪声与真启动之间的区别

因此，后续设计重点不应是“保留筛选器名字和参数”，而应是“保留筛选器背后的识别思想”。

### 5.2 不同筛选器在识别不同类型的机会

它们不是同一类模式的不同命名，而是在识别不同阶段、不同风险结构、不同量价行为的股票。

### 5.3 后续不应机械搬运

这些筛选器不应被默认原封不动塞进 engine 的某个环节。

正确做法应当是：

- 先理解其精神含义
- 再判断它更像 `M1`、`M2` 还是 `M3` 的资产
- 最后决定它以“事实、证据、规则还是实验资产”的形式保留

## 6. 各筛选器的精神含义

以下结论基于配置参数、runtime 条件描述、历史文档和系统现有说明做归纳。

### 6.1 杯柄 V4

精神含义：

- 在寻找“前期上涨后经过较长时间圆弧整理，并在柄部缩量后准备重新走强”的股票。

它试图抓的不是简单形态，而是：

- 趋势未死
- 结构没有被破坏
- 调整较充分
- 右侧确认临近

本质更接近：

- `结构蓄势完成后的再启动准备`

### 6.2 试盘线

精神含义：

- 在寻找“低位横盘 -> 高量试盘 -> 低量涨停/缩量回踩 -> 再次放量突破”的链式行为。

它试图识别的是：

- 主力试盘
- 控盘
- 洗筹
- 再启动

本质更接近：

- `主力试盘与控盘完成后的启动链`

### 6.3 二板回调

精神含义：

- 在寻找“情绪或强势启动已被市场看见后，经过短期回调再次给出参与窗口”的标的。

它试图识别的是：

- 情绪接力股
- 强势确认后的二次低风险上车点

本质更接近：

- `强势股的二次参与窗口`

### 6.4 每日冷热股

精神含义：

- 在寻找“当前最热/最冷、最有成交和关注度”的对象。

它不是完整结构筛选器，而更像：

- 市场热度
- 资金与关注度
- 风险偏好

本质更接近：

- `市场状态输入`

### 6.5 金凤凰

精神含义：

- 从参数结构看，它试图抓取一条包含“放量确认、再放量、地量回落、再次启动”的序列。

这说明它不只是静态 K 线形态，而是在观察：

- 强势出现
- 行为延续
- 中间压缩/回落
- 再启动

本质更接近：

- `结构-情绪-启动链中的强势延续模板`

### 6.6 银凤凰

精神含义：

- 与金凤凰相近，但更强调“缩量确认后再启动”的质量。

它试图区分的是：

- 真正回落后的蓄势
- 还是单纯冲高后的失真

本质更接近：

- `缩量确认后的再启动模板`

### 6.7 倍量阴

精神含义：

- 从“涨停、倍量阴、地量、启动日放量”等参数可见，它试图抓的是“强势动作后通过倍量阴洗筹，再在压缩后重新启动”的行为模式。

它在试图区分：

- 强势后的出货
- 与强势后的洗筹

本质更接近：

- `强势后洗筹再启动模板`

### 6.8 老鸭头（当前未在正式 screener 主链中启用）

精神含义：

- 在当前系统中，它更像一类 `周线结构确认`，用于说明中周期趋势是否仍健康、是否具备继续上攻的结构条件。

它强调的是：

- 周线均线结构
- 突破确认
- 不过度乖离

本质更接近：

- `中周期结构确认`

## 7. 精神含义分类

从精神层面看，当前筛选器可先分为两大类：

### 7.1 形态/结构型

- `cup_handle_v4`
- `shi_pan_xian`
- `jin_feng_huang`
- `yin_feng_huang`
- `zhang_ting_bei_liang_yin`
- `lao_ya_tou_zhou_xian`

它们共同关注：

- 结构是否完整
- 洗筹是否充分
- 启动是否真实
- 走势是否具备阶段性规律

### 7.2 市场热度/关注度型

- `daily_hot_cold`

它主要关注：

- 市场资金与关注度
- 情绪冷热
- 当前最受关注对象

### 7.3 情绪接力/阶段切入型

- `er_ban_hui_tiao`

它既不是单纯结构型，也不是纯市场热度型，而是更偏：

- 情绪延续后的再切入点识别

## 8. 当前 engine 吸收状态

### 8.1 已明确直接吸收

- `cup_handle_v4`
  - 通过 artifact 读取或 runtime 调用接入 engine
- `老鸭头相似逻辑`
  - 以 engine 内置周线结构确认方式存在

### 8.2 已碎片化吸收

以下经验语义，已经散落到 engine 内部逻辑中，但没有独立作为筛选器资产管理：

- 结构确认
- 波段不确定降权
- 低共振降权
- focus gate
- 跟随股降权
- tracking 观察保留
- 执行层再次闸门

### 8.3 尚未系统吸收

以下筛选器当前尚未以“完整精神模型”进入 engine 主链：

- `shi_pan_xian`
- `er_ban_hui_tiao`
- `jin_feng_huang`
- `yin_feng_huang`
- `zhang_ting_bei_liang_yin`
- `daily_hot_cold`（其市场热度思想仅有相似逻辑，并非同一实现链）

## 9. 推荐层级归属

这里的“归属”指精神资产的主要归位方向，而不是要求原代码必须整体搬迁。

### 9.1 更接近 M1 的资产

- `daily_hot_cold`

原因：

- 它更像市场热度、参与度、成交活跃度的事实输入或派生事实输入
- 不应直接作为最终交易真值，而应作为上游状态输入

### 9.2 更接近 M2 的资产

- `cup_handle_v4`
- `shi_pan_xian`
- `lao_ya_tou_zhou_xian`
- `jin_feng_huang`
- `yin_feng_huang`
- `zhang_ting_bei_liang_yin`

原因：

- 它们大多表达的是“当前处于什么结构阶段、是否完成蓄势、是否具备启动条件”
- 这些语义更接近周期与结构识别证据，而不是最终交易动作

### 9.3 更接近 M3 的资产

- `er_ban_hui_tiao`
- 以及部分由 M2 输出下沉后的 `再切入 / tracking 升级 / entry 窗口` 规则

原因：

- 它更像强势确认后的具体参与窗口判断
- 更接近“何时上车”而不是“是否处于某种结构阶段”

## 10. 后续设计纪律

基于本研究，后续 `M1/M2/M3` 设计必须遵守以下纪律：

### 10.1 不机械嫁接

- 不得默认将现有筛选器原封不动接入 engine。

### 10.2 先保留精神，再决定形式

- 先保留识别思想，再决定是进入：
  - `M1` 的事实/派生事实
  - `M2` 的结构证据
  - `M3` 的候选/entry 规则
  - 或保留为研究资产

### 10.3 不允许经验资产无声丢失

- 任何后续重构，如果导致某个筛选器精神被放弃，必须显式说明：
  - 放弃了什么
  - 为什么放弃
  - 由什么更高质量机制替代

### 10.4 不允许以“更先进架构”为名弱化团队经验

- 新架构必须吸收和提升这些经验资产，而不是用抽象概念把它们抹平。

## 11. 当前结论

本文冻结以下研究结论：

- 现有筛选器本质上是一组团队经验沉淀出的模式假设。
- 它们真正重要的是“精神含义”，而不是配置文件本身。
- 当前 lowfreq engine 只直接吸收了少部分筛选器，其余更多是碎片化吸收或尚未系统吸收。
- 后续设计必须先研究筛选器在识别什么，再决定它们归属于 `M1/M2/M3` 的哪一层。
- 任何未来重构都不得让这些经验资产无声丢失。

## 12. 不在本文档解决的问题

以下问题不在本文档中展开：

- 各筛选器的最终算法抽象方式
- 哪些筛选器将被正式纳入 benchmark
- 哪些筛选器将作为研究型资产长期保留
- `M2` 如何将这些结构证据统一表达成周期状态
- `M3` 如何将其中一部分转化为 tracking / entry 决策

这些内容将进入后续 `M1/M2/M3` 详细设计文档。
