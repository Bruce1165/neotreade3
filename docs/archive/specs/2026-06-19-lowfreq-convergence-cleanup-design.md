# 低频买入主链旧收敛层清理设计（设计定稿）

日期：2026-06-19  
范围：`focus gate` 之后的正式买入信号收敛层，以及仍在压制新逻辑释放的旧数量型/阈值型收敛器

## 1. 背景与目标

- 当前低频买入主链已经完成以下修正：
  - 正式链 / 回测链统一接入 5 日买入信号记忆窗口
  - `attention_score` 从硬门槛降为参考项
  - `方案 A` 已放宽：
    - 热门板块候选数
    - 跨板块扫描深度
    - 跨板块 `Wave 1`
  - 跨板块名额已从 `1` 放宽到 `2`
- 18 个月回测已恢复到：
  - `5` 笔交易
  - `12.78%` 收益
  - `4.04%` 最大回撤
- 但交易仍高度集中在 `2026-05 ~ 2026-06`。

这说明：
- 新逻辑已经开始生效，但还没有在整段历史窗口全面释放。
- 当前瓶颈并不一定只来自旧收敛层，也可能来自历史证据链不完整。

目标：
- 在历史证据链补齐后，按分层方式清理旧收敛层，而不是一次性拆空所有质量控制。
- 让上游新逻辑不再被下游重复语义的旧硬截断覆盖。
- 保留必要的质量控制，但把与新逻辑冲突的旧硬闸改成更弱的排序项或直接移除。

## 2. 当前问题拆解

当前正式买入链可以拆成四层：

1. 基础打分
   - 基本面
   - 价格位置
   - 波浪阶段
   - 同频共振
   - 温和放量
   - 5 日涨幅
   - 均线趋势
   - 市值偏好
2. 角色收敛
   - 板块内强度排序
   - 龙头 / 中军 / 跟随角色识别
3. `focus gate`
   - 第二大类 / AI / 渗透率 / 配置证据 / 龙头语义
4. 正式买入信号收敛
   - 热门板块入口限制
   - 板块内数量限制
   - 跨板块扫描深度与名额
   - 跨板块额外分差
   - 共振硬门槛

本次要清理的是第 4 层为主，第 3 层为辅。

## 3. 现状证据

### 3.1 热门板块候选入口仍是旧数量型截断

- `get_sector_candidates()` 的底层原始样本仍只查 `LIMIT 20`。
- 这代表即使某板块内存在更靠后的合格候选，也在进入评分前被截断。

### 3.2 热门板块正式信号仍受双重硬门槛

- 进入 `buy_signals` 前，热门板块候选仍必须同时满足：
  - `buy_score >= BUY_THRESHOLD`
  - `sector_resonance >= MIN_RESONANCE`
  - `role != 跟随`

### 3.3 跨板块分支仍保留强收敛

- 当前跨板块仍受：
  - `CROSS_SECTOR_MAX_SIGNALS`
  - `CROSS_SECTOR_SCORE_MARGIN`
  - `MIN_RESONANCE`
  - `role != 跟随`
  - `Wave` 限制
- 这条分支已被多次放宽，但仍是“高分少量补位”语义，而不是新逻辑的主释放通道。

### 3.4 数据缺口会放大旧收敛层的表面严厉程度

- 当配置证据链缺失时，上游通过率本来就会降低。
- 此时继续只看第 4 层压缩效果，容易误判为“旧收敛层过紧”，而忽略“上游已缺样本”。

## 4. 方案比较

### 4.1 方案 A：继续只调数量上限

- 继续提高：
  - 热门板块入口深度
  - 跨板块名额
  - 跨板块扫描深度

优点：
- 改动最小。
- 风险低。

缺点：
- 只解决“看得更多”，不解决“硬门槛仍重复压缩”。

### 4.2 方案 B：把重复语义的硬过滤改成排序项

- 保留核心质量控制，但把部分硬闸改成排序惩罚或软约束。
- 例如：
  - 共振不足不直接过滤，而是排序降权
  - 跨板块额外分差不作为硬门槛，而是排序加权

优点：
- 更符合“新主逻辑先决定能否入场，旧逻辑只做排序微调”的方向。

缺点：
- 需要同时调整评分与最终收敛规则，改动复杂度较高。

### 4.3 方案 C：分阶段清理

- 第一阶段：
  - 只清数量型、重复语义最强的旧硬截断
- 第二阶段：
  - 把部分质量控制从硬闸改为排序项
- 第三阶段：
  - 仅保留风险控制和明显有价值的质量底线

优点：
- 便于按证据逐步推进。
- 每阶段都能独立回测与归因。

缺点：
- 比一次性拆除更慢。

结论：
- 本次采用方案 C。

## 5. 设计原则

- 不在历史证据链未恢复前，直接做最终版大拆。
- 优先移除与新逻辑重复表达的旧数量型截断。
- 风险控制优先保留，语义重复的质量控制优先降级。
- 每一轮清理都必须单独回测，避免多个变量同时变化。
- 如果某项约束仍能显著降低回撤且不明显压制交易数，应优先保留。

## 6. 分阶段清理设计

### 6.1 第一阶段：清数量型旧硬截断

目的：
- 先释放“新逻辑本来已经选出的票”，避免下游用老式数量上限继续压缩。

范围：
- 热门板块原始候选入口：
  - 重新评估 `LIMIT 20` 是否仍合理
- 板块内候选请求数：
  - 继续放宽或改成与热度动态关联
- 跨板块名额：
  - 重新评估 `CROSS_SECTOR_MAX_SIGNALS`

边界：
- 这一阶段不动 `BUY_THRESHOLD`
- 不动卖出链
- 不直接去掉角色约束

验收：
- `focus gate` 通过票里，被数量上限直接截掉的比例明显下降。

### 6.2 第二阶段：把重复语义的硬质量闸改成排序项

候选对象：

#### 共振约束

- 当前热门板块和跨板块都要求：
  - `sector_resonance >= MIN_RESONANCE`
- 这和上游“主线命中 + 龙头 + 配置证据”存在部分重复表达。

设计：
- 不直接删除共振。
- 改为：
  - 高共振优先排序
  - 低共振降权
  - 只对极低共振保留硬过滤

#### 跨板块额外分差

- 当前跨板块要求：
  - `buy_score >= BUY_THRESHOLD + CROSS_SECTOR_SCORE_MARGIN`

设计：
- 改为跨板块排序惩罚，而不是硬挡。
- 让跨板块票与热门板块票一起竞争最终名额时自然处于次优位，而不是预先剔除。

#### `Wave` 限制

- 当前已经从只留 `Wave 3` 放宽到可含 `Wave 1`。
- 下一步不建议再把 `Wave` 全部放开。

设计：
- 保留 `Wave 1 / Wave 3` 主通道。
- 其他波段继续挡住，避免策略语义漂移。

### 6.3 第三阶段：清理重复的跨层语义

判断标准：
- 若上游已明确表达：
  - 主线
  - 龙头
  - 配置高配
  - 结构确认
- 下游就不应再用近似语义重复收缩。

候选清理对象：
- 板块内“只看涨幅靠前”的早期入口
- 与 `focus gate` 含义高度重叠的额外偏好项

## 7. 明确保留项

以下内容在当前设计中不作为优先清理对象：

- `role != 跟随`
  - 当前正式主链仍明确要求细分赛道龙头优先。
- `BUY_THRESHOLD`
  - 在已有证据下，它不是当前首要瓶颈。
- 卖出链
  - 当前卖出结构已经恢复，不应被买入链改造牵连。
- `Wave 1 / Wave 3` 主通道
  - 仍然符合当前策略语义。

## 8. 与数据补齐的耦合关系

- 若 `fund_portfolios / etf_basic_info / etf_index_basic / index_weights` 未补齐：
  - 不进入第二阶段及之后的大幅清理。
- 若财报公告时点未修正：
  - 不用长窗结果对“质量控制过严”作最终判断。
- 只有在历史证据链恢复后，旧收敛层清理才有可靠归因基础。

## 9. 执行顺序

### 9.1 第一步：完成历史证据补齐

- 以《低频回测历史证据补齐设计》为前置条件。

### 9.2 第二步：重新跑 18 个月回测

- 观察：
  - 交易数
  - 交易时间分布
  - `focus gate` 通过数与最终成交数之间的落差

### 9.3 第三步：第一阶段清理

- 只改数量型旧硬截断。
- 改完即回测。

### 9.4 第四步：第二阶段清理

- 在数据与第一阶段回测都确认后，再把部分硬质量闸降级为排序项。

### 9.5 第五步：视结果决定是否进入第三阶段

- 若交易数已恢复且回撤可接受，则不继续大拆。
- 若交易数仍显著偏低，才进入更深层的跨层语义清理。

## 10. 验收口径

### 10.1 结构验收

- `focus gate` 之后，不再存在明显重复语义的数量型硬截断压缩主样本。
- 旧收敛层对新逻辑的作用从“覆盖/否决”变为“排序/微调”。

### 10.2 回测验收

- 交易数相对当前 `5` 笔有实质恢复。
- 交易分布不再只集中在 `2026-05 ~ 2026-06`。
- 最大回撤不出现不可接受劣化。
- 卖出结构仍保留：
  - 止损
  - 分批止盈
  - 目标达成
  - 回测结束平仓

## 11. 风险与边界

- 若在数据未补齐前继续大拆旧链路，可能把缺数据导致的空窗误判为模型缺陷。
- 若把共振和跨板块额外分差同时一次性全部移除，可能使信号质量快速劣化，回撤失真。
- 本设计不改变策略大方向：
  - 仍以细分赛道龙头为主
  - 仍以 `Wave 1 / Wave 3` 为主通道
  - 仍保留卖出链现有修复成果

## 12. 成功标准

- 历史证据链恢复后，新逻辑能够在长窗内连续释放，而不是只在最近两个月局部生效。
- 旧收敛层从“决定能不能买”降级为“调节先后顺序”。
- 长窗回测可以明确回答：
  - 数据补齐后是否已足够恢复交易
  - 如果仍不足，究竟是哪一层旧收敛还在压制新逻辑
