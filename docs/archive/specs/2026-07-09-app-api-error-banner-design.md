# App 全局 API 错误横幅设计

日期：2026-07-09

## 1. 目标

本切片只解决一个问题：

当前端任意页面触发全局 `neotrade3:api-error` 事件，且错误语义表明本地 API 不可达或请求超时时，`App` 壳层需要展示统一、明确、可关闭的全局错误横幅。

本切片的目标不是重做页面级错误处理，而是为应用壳层补上一条稳定的全局环境异常提示链路。

## 2. 范围

包含：

- `neotrade3-dashboard/src/App.jsx`
- `neotrade3-dashboard/src/App.test.jsx` 中与全局 API 错误横幅直接相关的最小变更
- 或一个新的聚焦测试载体（如果比修改 `App.test.jsx` 更利于保持边界）

不包含：

- 侧边栏信息架构调整
- `OpsCenter` 接入
- Header 环境文案改写
- `Tushare 黄旗` 的展示优化
- 页面内部错误态改写
- 重试、队列、自动消失、本地持久化等增强交互

## 3. 现有证据

当前工作树已存在一条未提交的 `App` 漂移线，包含：

- 一个环境异常判定函数
- 一个全局 API 错误横幅组件
- 一个通过派发 `neotrade3:api-error` 进行验证的测试雏形

这说明本切片不是凭空发明的新需求，而是对现有可验证改动线进行边界收口。

## 4. 职责归属

全局 API 错误横幅的职责属于 `App` 壳层，而不属于任一单页。

原因：

- 事件来源是全局浏览器事件
- 受影响页面并不固定
- 提示语义是“当前开发环境/接口连通性异常”，而不是“某一个页面的业务错误”

因此：

- 页面继续负责各自局部错误态
- `App` 负责统一承接全局环境异常提示

## 5. 事件判定设计

### 5.1 事件来源

横幅通过以下方式监听全局错误事件：

- `window.addEventListener('neotrade3:api-error', handler)`

组件只消费事件 `detail` 中已经存在的字段，不在本切片定义新的事件协议。

### 5.2 环境异常判定

命中以下任一条件时，视为“开发环境未连接”：

- `detail.code === 'api_unreachable'`
- `detail.code === 'api_timeout'`
- `detail.message` 包含 `后端不可达`
- `detail.message` 包含 `请求超时`

该判定逻辑收敛在单一函数中，例如：

- `isEnvironmentUnavailable(apiError)`

目的是避免将相同判断散落到多个渲染分支。

## 6. 横幅行为设计

### 6.1 展示行为

当命中环境异常判定时，横幅显示：

- 标题语义：`开发环境未连接：`
- 主文案：`本地 API 服务暂未连接，页面会先展示占位信息。`

当错误存在但不属于环境异常时，可沿用同一个横幅容器显示普通“接口失败告警”语义。

本切片的重点验证路径是环境异常分支。

### 6.2 上下文信息

当事件 detail 中存在以下字段时，横幅允许通过折叠区展示：

- `endpoint`
- `status`
- `code`
- `happenedAt`

这些字段属于辅助诊断信息：

- 默认不强制展开
- 不参与主文案生成
- 仅做只读呈现

### 6.3 关闭行为

横幅必须支持手动关闭。

关闭行为只影响当前组件内存状态：

- 不写入 localStorage
- 不做跨刷新持久化
- 不做错误历史队列

再次收到新的全局 API 错误事件时，可以重新显示。

## 7. 非目标

本切片明确不处理：

- 多条错误队列
- 自动消失
- 页面级错误态去重
- 错误聚合中心
- 服务拉起、重试或自修复动作

本切片只负责“统一提示”，不负责“统一修复”。

## 8. 测试设计

测试应优先验证“壳层事件 -> 横幅渲染”链路，而不是顺手把 `App` 全部共享 copy 重测一遍。

推荐的最小测试集合：

1. 默认不显示全局 API 错误横幅
2. 当派发 `neotrade3:api-error` 且命中环境异常规则时，显示：
   - `开发环境未连接：`
   - `本地 API 服务暂未连接，页面会先展示占位信息。`
   - 可展开的辅助诊断信息入口

如果 `App.test.jsx` 当前过于混杂，更推荐新建聚焦测试文件，例如：

- `neotrade3-dashboard/src/App.apiErrorBanner.test.jsx`

## 9. 提交边界

目标实现提交只允许包含：

- `App.jsx` 中的环境异常判定函数
- `App.jsx` 中的全局 API 错误横幅组件
- `App.jsx` 中的横幅挂载点
- 对应的聚焦测试文件，或 `App.test.jsx` 的最小相关 hunk

必须排除：

- `OpsCenter` import / route
- 导航项标签与导航结构调整
- Header 环境文案改写
- `GlobalBanner` 的 `details` 展开优化

## 10. 验证要求

预期最小验证闭环：

- 运行全局 API 错误横幅聚焦测试
- 如触及 `App.test.jsx`，仅补跑相关 `App` 测试
- 使用精确暂存或索引快照，确保提交中不带其他 `App` 漂移

## 11. 结论

本切片的本质不是优化 `App` 外壳视觉，而是为当前前端壳层补上一条清晰、统一、可验证的全局环境异常提示链。

只有先把这条线从当前 `App` 混杂漂移中拆出来，后续导航 IA、环境文案、黄旗信息展开等其他 `App` 主题线才能继续按窄切片推进。
