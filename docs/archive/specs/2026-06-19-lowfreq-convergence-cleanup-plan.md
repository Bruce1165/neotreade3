# 低频买入主链旧收敛层清理实施计划

日期：2026-06-19  
对应设计：`docs/superpowers/specs/2026-06-19-lowfreq-convergence-cleanup-design.md`

## 1. 目标

按已确认设计，在历史证据链补齐之后，分阶段清理当前正式买入主链中仍在压制新逻辑释放的旧收敛层：

- 第一阶段：先清数量型旧硬截断
- 第二阶段：把部分重复语义的硬质量闸改成排序项
- 第三阶段：仅在必要时清理更深层重复语义

本计划不在数据证据链未恢复前直接大拆旧逻辑，也不改动当前卖出链。

## 2. 当前落地约束

### 2.1 已有放宽成果

- `attention_score` 已降为参考项
- 热门板块候选数已放宽
- 跨板块扫描深度已放宽
- 跨板块 `Wave 1` 已放行
- `CROSS_SECTOR_MAX_SIGNALS` 已从 `1` 放宽到 `2`

### 2.2 当前仍在生效的旧收敛器

- 热门板块底层原始样本入口仍受 `LIMIT 20` 限制
- 热门板块正式信号仍受：
  - `BUY_THRESHOLD`
  - `MIN_RESONANCE`
  - `role != 跟随`
- 跨板块分支仍受：
  - `CROSS_SECTOR_MAX_SIGNALS`
  - `CROSS_SECTOR_SCORE_MARGIN`
  - `MIN_RESONANCE`
  - `Wave` 限制

### 2.3 当前关键前置条件

- 只有在历史证据补齐计划完成后，才能对旧收敛层做可靠归因。
- 如果数据仍缺失，继续大幅清链路会把“缺数据导致的无票”误判成“旧链路太紧”。

## 3. 实施原则

- 先恢复数据证据，再清收敛层。
- 每一轮只改一层，不同时改多个收敛变量。
- 每一轮清理后都要跑定向测试和长窗回测。
- 保留当前卖出链与核心策略方向：
  - 细分赛道龙头优先
  - `Wave 1 / Wave 3` 主通道
  - 已恢复的止损 / 止盈结构
- `BUY_THRESHOLD` 本轮不作为第一优先级改动对象。

## 4. 实施分解

### Phase 0：前置验证

目标：
- 确认当前已具备进入旧收敛层清理的前置条件。

任务：
- 核对历史证据补齐计划执行结果
- 重跑长窗回测
- 输出当前基线：
  - 交易数
  - 交易时间分布
  - 近期集中度
  - `focus gate` 通过数与最终买入数的落差

完成判定：
- 能明确“数据恢复后，旧收敛层仍然是主瓶颈”

### Phase 1：数量型旧硬截断清理

目标：
- 先清掉最明显的数量型压缩，释放本来已经通过上游新逻辑的候选。

任务：
- 重新评估并调整：
  - 热门板块底层原始样本 `LIMIT 20`
  - 热门板块候选请求数
  - 跨板块最大放行数
- 保持不变：
  - `BUY_THRESHOLD`
  - `MIN_RESONANCE`
  - `role != 跟随`
  - 卖出链

测试要求：
- 新增或调整针对数量型限制的回归测试
- 验证：
  - 候选入口深度变化已生效
  - 不会无限放大跨板块信号

完成判定：
- `focus gate` 通过后、被数量型限制直接挡掉的比例明显下降

### Phase 2：硬质量闸降级为排序项

目标：
- 把与新逻辑重复表达的部分硬闸，从“能不能买”改为“排在前还是后”。

任务：
- 评估并逐项改造：
  - `MIN_RESONANCE`
  - `CROSS_SECTOR_SCORE_MARGIN`
- 改造方式：
  - 优先级惩罚
  - 排序降权
  - 极低值才继续保留硬挡

测试要求：
- 新增排序行为测试
- 明确验证：
  - 低共振不一定被直接过滤
  - 跨板块票在排序中落后，而不是预先被删除

完成判定：
- 旧质量闸不再重复否决上游新逻辑已通过的候选

### Phase 3：跨层重复语义清理

目标：
- 只在前两阶段后仍然明显压制交易数时，才进入更深层清理。

任务：
- 识别与 `focus gate` 明显重复的下游偏好项
- 评估是否保留：
  - 板块内过早的早期入口偏好
  - 与“主线 / 龙头 / 配置证据”语义重叠的二次收敛

边界：
- 本阶段为条件触发，不默认执行

完成判定：
- 下游收敛层不再覆盖上游主逻辑语义

## 6. 测试与验收

### 6.1 定向测试

- 数量型限制变化有测试覆盖
- 排序降级逻辑有测试覆盖
- 现有：
  - 卖出链测试
  - `focus gate` 测试
  - 信号记忆窗口测试
  不能回退

### 6.2 回测验收

每一阶段都要至少重跑：

- 近端窗口
  - 观察交易结构是否被破坏
- 18 个月长窗
  - 观察交易数
  - 观察交易分布
  - 观察收益与回撤

### 6.3 成功判定

- 相比当前 `5` 笔交易，交易数有实质提升
- 交易分布不再只集中于 `2026-05 ~ 2026-06`
- 卖出结构仍保留：
  - 止损
  - 分批止盈
  - 目标达成
  - 回测结束平仓
- 最大回撤没有不可接受恶化

## 7. 风险与控制

### 风险 1：数量型放宽后短期噪声票增加

控制：
- 第一阶段只动数量上限，不动分数和角色
- 放宽后立即跑近端与长窗回测

### 风险 2：把质量闸降级过快导致回撤失真

控制：
- `MIN_RESONANCE` 与 `CROSS_SECTOR_SCORE_MARGIN` 不同时一次性完全取消
- 先改排序，再看结果

### 风险 3：在数据未完全恢复前做出错误归因

控制：
- 本计划明确依赖历史证据补齐计划完成
- 如果数据链未恢复，本计划不进入 Phase 2/3
