# 低频长持优先主框架设计

日期：2026-06-20  
范围：`lowfreq_engine_v16_advanced.py` 对应低频模型的正式离场主链、系统性见顶判断框架、龙头增强持有原则、回测验收口径

## 1. 背景

最近几轮低频模型调整已经证明两个事实：

- 入口覆盖问题可以通过放宽市值带显著改善
- 容量与成交口径问题可以通过 `5000万 + 8仓 + 仅一字板阻断` 显著改善

但离场层仍然没有进入真正的低频框架。

最近一轮 `anti-noise` A/B 实验虽然在组合结果上有所改善：

- 总收益从 `68.99%` 升到 `73.10%`
- 最大回撤从 `16.81%` 降到 `16.31%`
- 交易数从 `520` 降到 `264`

但核心问题仍然存在：

- 平均持仓天数仍明显偏短，尚未进入“低频长持”区间
- 大量离场仍是在试图处理中途震荡，而不是只处理真正错判或系统性见顶
- `Top200` 捕获口径下，`bought_count` 与 `held_to_top_count` 反而下降，说明当前离场演化仍不在正确的大框架上

用户已明确要求先落实大的逻辑框架，再谈细节优化。用户确认的方向是：

- 低频交易应以长持有为中心，而不是以频繁处理过程波动为中心
- 建仓后默认忽略过程中的个股震荡
- 个股侧只保留买入价 `-5%` 的硬止损阈值
- 主动离场主要由 `大盘见顶` 与 `板块见顶` 决定
- 平均持仓天数目标应靠近 `2-3个月`，大致即 `40-60` 个交易日
- 若持仓既是牛股又是板块龙头，应更坚定持有

## 2. 目标

本轮设计的唯一目标是把低频模型的离场逻辑重构到“长持优先”框架上。

本轮要解决：

- 明确正式离场主链只剩哪些出口
- 明确 `大盘见顶` 与 `板块见顶` 的职责、优先级与状态机语义
- 明确“核心牛股 + 板块龙头”为什么以及如何使用更高确认门槛
- 明确哪些旧规则必须退出主链，哪些只能降级为证据或审计信息
- 明确后续实现与回测应以什么口径验证“框架已正确”

本轮不解决：

- `大盘见顶` 与 `板块见顶` 的具体参数数值
- 某个单一因子的数学优化
- 买入逻辑重构
- 本金按需投入统计口径
- 分层减仓
- 仓位动态扩张或收缩策略
- 新的外部数据源接入

## 3. 方案比较

### 3.1 方案 A：长持优先主框架

- 建仓后默认持有
- 个股侧只保留买入价 `-5%` 硬止损
- 其余主动离场只允许来自 `大盘见顶确认` 或 `板块见顶确认`
- 所有个股过程震荡型规则退出正式卖出主链

优点：

- 与用户确认的低频哲学完全一致
- 能从根本上压缩交易频次
- 能避免在错误框架里不断微调离场细节

缺点：

- 会显著改变现有卖出结构
- 真正的系统性见顶判断必须做得更严谨，否则会出现迟钝风险

### 3.2 方案 B：保留个股回撤，但下调优先级

- 继续保留个股回撤类卖出
- 只是不再让其压在最前面

优点：

- 对现有代码和行为改动较小

缺点：

- 本质上仍然在处理中途震荡
- 很容易重新滑回“信号多就卖”的旧路径
- 不符合“低频长持”框架

### 3.3 方案 C：锁仓窗口

- 建仓后前若干交易日仅允许 `-5%` 止损
- 锁仓结束后再恢复系统性卖出链

优点：

- 容易快速拉长平均持有时间

缺点：

- 它是时间硬约束，不是逻辑框架
- 用户已明确 `20-60` 天应先作为结果目标，而不是硬门槛

结论：

- 采用 `方案 A：长持优先主框架`

## 4. 核心设计原则

- `持有优先`：建仓后默认继续持有，不因个股中途震荡、回撤、短弱主动离场
- `个股唯一硬退出`：个股侧只保留买入价 `-5%` 硬止损
- `系统性退出优先`：除 `-5%` 外，主动离场只允许来自 `大盘见顶确认` 和 `板块见顶确认`
- `确认重于预判`：单日异动不能直接卖出，必须经过观察、复核、确认
- `龙头更高门槛`：当持仓同时满足“核心牛股 + 板块龙头”时，系统性见顶确认门槛应高于普通持仓
- `框架先于收益`：先验证是否真正进入低频长持框架，再评价收益表现
- `旧规则降级`：旧有个股回撤、短期弱化、浮盈保护等信号如仍保留，只能作为观察证据或审计信息

## 5. 正式离场主链

本轮设计要求将正式离场主链压缩为三个出口：

1. 买入价 `-5%` 硬止损
2. `大盘见顶确认`
3. `板块见顶确认`

其中语义定义如下：

### 5.1 买入价 `-5%` 硬止损

- 这是个股级唯一正式硬退出
- 它表示这笔交易被证伪
- 它不依赖任何观察态或复核态
- 一旦满足，立即执行

### 5.2 大盘见顶确认

- 这是组合级总开关
- 回答的问题是：市场整体是否已经不适合继续重仓持有
- 一旦确认，应作用于全组合
- 若与板块见顶同日同时成立，以 `大盘见顶` 记账

### 5.3 板块见顶确认

- 这是主线级分支开关
- 回答的问题是：该持仓所属主线是否已经完成主升并进入系统性退潮
- 一旦确认，只作用于对应板块持仓
- 不应影响其他仍处强势主线的仓位

### 5.4 优先级

正式优先级固定为：

1. 买入价 `-5%` 硬止损
2. `大盘见顶确认`
3. `板块见顶确认`

含义：

- `-5%` 是单笔交易证伪，必须最先执行
- `大盘见顶` 是更高层级环境反转
- `板块见顶` 是局部主线退潮

## 6. 大盘见顶与板块见顶的判断框架

`大盘见顶` 和 `板块见顶` 都不能再被定义成“单日触发即卖”的事件。

它们都应被定义为：

- `系统性风险反转的确认过程`

### 6.1 总体结构

两者统一采用：

- `观察态 -> 复核态 -> 确认离场`

### 6.2 判断原则

- 单一信号无权卖出
- 短期回调默认容忍
- 多方法证据优先
- 修复失败比单次破坏更重要
- 确认重于预判

### 6.3 方法族

`大盘见顶` 与 `板块见顶` 都必须由多组方法共同提供证据，不能由单一因子拥有卖出权。

方法族至少包括：

- `价格趋势组`
  - 判断趋势是否真正破坏，而非普通回踩
- `广度强弱组`
  - 判断是否从普强转向系统性退潮
- `领涨核心组`
  - 判断核心带动品种是否失去主升地位
- `量能与修复组`
  - 判断回调是否演化成修复失败后的结构反转

### 6.4 证据分层

观察证据：

- 单次趋势破坏
- 首次广度恶化
- 个别核心异常
- 单次量价修复不佳

确认证据：

- 同类证据持续恶化
- 不同方法族开始共振
- 修复失败
- 广度退潮与领涨核心退潮同步出现

正式定义可压缩为：

- `见顶 = 多方法证据在时间上持续、在结构上共振、且修复失败后的确认结果`

## 7. 状态机

状态机只服务于 `大盘见顶` 与 `板块见顶`。买入价 `-5%` 硬止损不进入该状态机。

### 7.1 持有态

- 默认状态
- 继续持有
- 个股过程震荡、普通回撤、短期走弱都不能直接打出仓位

### 7.2 观察态

- 出现疑似顶部迹象
- 只能记录风险并继续跟踪
- 不允许卖出
- 若后续修复成功，回到持有态

### 7.3 复核态

- 不再是单点异动
- 多组证据开始同步走弱，或同类证据持续恶化
- 仍然默认继续持有
- 只有继续恶化并达到确认标准，才允许升级为离场

### 7.4 确认离场

- 多方法证据共振
- 且修复失败
- 才执行 `大盘见顶离场` 或 `板块见顶离场`

### 7.5 状态机硬约束

- 单日异动只能进入观察态，不能直接卖出
- 复核阶段默认继续持有，不能因为不确定性提前卖出
- 系统性离场必须是确认结果，而不是恐慌型预判

## 8. 龙头增强持有原则

### 8.1 原则

若持仓同时满足：

- `核心牛股`
- `板块龙头`

则系统性见顶确认门槛必须高于普通持仓。

这不是“龙头永远不卖”，而是：

- 龙头不能按普通股的敏感度去卖

### 8.2 为什么必须提高确认门槛

- 龙头往往最先启动，也最容易在主升中出现剧烈震荡
- 当它同时是板块龙头时，中途回调很可能仍是洗盘或主升中的波动
- 如果使用与普通股相同的确认敏感度，最容易把真正该拿住的仓位洗掉

### 8.3 进入条件

`龙头增强持有` 只对“核心牛股 + 板块龙头”的交集生效，不是二选一。

#### 核心牛股

必要判断方向：

- 主升持续性
- 趋势独立性
- 阶段辨识度

辅助强化方向：

- 启动早
- 新高能力强
- 回撤修复快

本质定义：

- `持续主升 + 强于同类 + 修复能力强`

#### 板块龙头

必要判断方向：

- 方向锚
- 涨幅锚
- 带动锚

辅助强化方向：

- 启动领先
- 持续活跃
- 同板块相对优势稳定

本质定义：

- `能代表主线、能带动同板块、能在较长阶段维持核心地位`

### 8.4 必须排除的对象

以下对象不得进入 `龙头增强持有`：

- 补涨股
- 跟风股
- 脉冲股
- 普通强势股

### 8.5 执行语义

对进入 `龙头增强持有` 的持仓：

- 观察态更难升级到复核态
- 复核态更难升级到确认离场
- 需要更强、更持续、更多组共振证据才允许认定真正见顶

## 9. 旧规则的重构归位

现有旧规则必须被重新划分为三类：

1. 正式卖出规则
2. 观察 / 复核证据
3. 审计信息

### 9.1 保留在正式主链的规则

- 买入价 `-5%` 硬止损
- `大盘见顶确认`
- `板块见顶确认`

### 9.2 必须移出正式主链的规则

- 个股回撤止盈 / 止损类规则，例如当前 `stock_drawdown`
- 个股短期弱化类规则
- 单日 `market_top` / 单日 `sector_top`
- 以锁定浮盈为目的的主动卖出

这些规则不得再处于“满足即卖”的位置。

### 9.3 可降级为观察或复核证据的规则

- 个股高位回撤
- 短期趋势破坏
- 市场 / 板块单次走弱
- 量价修复失败

这些规则只负责提供证据，不负责执行卖出。

### 9.4 仅保留为审计信息的内容

- 某次短时异动
- 某天极端波动
- 某个单点指标的短期恶化
- 只在事后解释中有意义、但不应影响主链的碎片信号

## 10. 验收口径

本轮框架重构不能以总收益为第一指标，必须先验证框架是否正确。

### 10.1 框架型指标

最高优先级看：

- `交易数` 是否显著下降
- `平均持仓天数` 是否明显上升并向 `40-60` 个交易日靠近
- `持仓天数分布` 是否摆脱大量 `1-10` 天即卖出的结构
- `系统性离场占比` 是否上升并成为主要离场来源

### 10.2 捕获型指标

重点看：

- `Top200 bought_count`
- `held_to_top_count`
- 提前离场样本数
- “核心牛股 + 板块龙头”样本的持有质量

### 10.3 结构型指标

必须同步监控：

- 满仓阻塞占比
- 不同卖出原因分布
- 板块集中度
- 组合周转率

### 10.4 结果型指标

继续观察，但优先级后置：

- 总收益
- 最大回撤
- 胜率
- 盈亏比

解释原则：

- 收益变好，不代表框架已经正确
- 收益暂时变差，也不代表框架错误
- 只要框架型指标显著改善，且结果型指标没有灾难性恶化，就说明方向值得继续迭代

### 10.5 第一轮验收方向

第一轮不写死数值参数，先看方向性要求是否成立：

- 交易数明显下降
- 平均持仓明显上升并向 `40-60` 个交易日靠近
- 系统性离场成为主因
- 提前离场的牛股样本减少
- 龙头样本持有质量改善

## 11. 实施边界

后续实现必须遵守以下边界：

- 先重构正式离场主链，再做参数优化
- 先让旧规则退出主链，再讨论它们是否保留为证据
- 先实现可审计状态机，再做见顶阈值微调
- 先验证“低频长持框架是否成立”，再评价收益

## 12. 结论

本轮设计的本质不是继续优化某个局部卖点，而是把低频模型的离场哲学彻底改成：

- `买入后默认持有`
- `个股只在被证伪时止损`
- `系统性风险只在确认见顶后离场`
- `对核心牛股 + 板块龙头给予更高确认门槛`

这一定义为后续实现提供了清晰边界：

- 正式卖出链只剩三个出口
- 所有过程震荡类规则退出正式主链
- `大盘见顶` 与 `板块见顶` 必须改造成“多方法综合判断 + 延迟确认”的系统性状态机
- 任何后续细节优化都必须服务于这个长持优先框架，而不能再次破坏它
