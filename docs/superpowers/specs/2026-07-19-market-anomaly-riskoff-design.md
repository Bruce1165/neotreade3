# 2026-07-19 市场整体异动检测与 Risk-Off 设计（v0）

## 1. 背景与目标

本设计用于为 LowFreq V16 增加“市场整体异动”能力：当出现疑似系统性撤资/一致性砸盘等非正常市场波动时，模型必须立刻触发系统性退出，并进入观察阶段，直到市场稳定、人气（资本）回归后，周期得以继续。

本设计的核心目标不是解释“机构意图”，而是把“市场整体异动”形式化为可复算、可审计、可回放的统计信号，并将其接入现有系统性退出链路，形成可验证闭环。

## 2. 范围冻结

### 2.1 本轮范围

本轮只做：

1. 定义 v0 异动触发口径（可复算、可审计）
2. 定义触发后的行为语义（risk-off：全清仓 + 观察锁）
3. 定义“两段式恢复”的结构与恢复期门控策略（不固化恢复参数的数值）
4. 定义审计字段与证据束落盘要求（用于离线归因与回测对照）

### 2.2 本轮边界外

以下内容不纳入本轮：

- 不引入外部指数/宏观数据源作为硬依赖（v0 只用 `daily_prices` 统计）
- 不引入“机构行为归因”类语义（不可直接观测）
- 不固化“恢复到可开仓”的具体天数参数（需后续基于标注与回测校准）
- 不在本设计阶段直接改动引擎代码（先规格化，再进入实现计划）

## 3. 数据与指标口径（可复算）

### 3.1 数据源

v0 仅消费 `daily_prices`（事实层数据），按单交易日聚合全市场横截面统计：

- `pct_change`：当日涨跌幅（百分数）
- `amount`：成交额（以库内口径为准）

### 3.2 指标定义

对每个交易日 `T`，定义：

- `down5_ratio(T)`：全市场 `pct_change <= -5` 的股票占比
- `down7_ratio(T)`：全市场 `pct_change <= -7` 的股票占比
- `pct_mean(T)`：全市场 `pct_change` 横截面平均值
- `limit_down_proxy_ratio(T)`：全市场 `pct_change <= -9.5` 的股票占比（作为跌停近似代理指标，仅用于审计与增强项）
- `total_amount(T)`：全市场 `SUM(amount)`（仅用于审计与增强项）
- `amount_ratio_to_ma5(T)`：`total_amount(T) / MA5(total_amount)`（仅用于审计与增强项）

本轮将 `limit_down_proxy_ratio/total_amount/amount_ratio_to_ma5` 固化为“证据束字段”，但不纳入 v0 触发条件。

## 5. v0 触发口径（冻结）

### 5.1 触发规则

定义 `market_anomaly_hit_v0(T)`：

当且仅当同时满足：

- `down5_ratio(T) >= 0.175`
- `down7_ratio(T) >= 0.06`
- `pct_mean(T) <= -1.5`

则 `market_anomaly_hit_v0(T) = True`，否则为 `False`。

### 5.2 v0 解释文本（审计要求）

当 `market_anomaly_hit_v0(T)=True`，必须输出可解释 reasons，至少包含：

- `capture-first: 市场整体异动 v0 命中（down5>=17.5%, down7>=6.0%, pct_mean<=-1.5%）`

并同时落盘 `anomaly_snapshot`，包含 3 个阈值字段与增强字段（第 3.2 节）。

## 6. 行为语义：Risk-Off（全清仓 + 观察锁）

当交易日 `T` 命中 `market_anomaly_hit_v0(T)=True`：

1. 当日立刻触发系统性退出（scope=market，portfolio 级全清仓）
2. 进入观察阶段（observe lock）：在观察锁解除前，禁止新增开仓

本设计不要求“先观察再确认才清仓”；命中即清仓是硬约束。

## 7. 恢复策略：两段式恢复（结构冻结，参数不固化）

### 7.1 第一段：解除全清仓锁

当连续 `K` 个交易日满足 `market_anomaly_hit_v0=False`，解除“全清仓锁”，从“严格锁定”进入“恢复期”。

说明：

- `K` 作为参数存在，但本轮不固化数值；后续通过标注与回测校准。

### 7.2 第二段：恢复期（更高门槛 + 严格限频）

解除全清仓锁后，进入恢复期（持续 `R` 个交易日），期间必须同时生效：

- 更高门槛：提高入场筛选门槛（例如提高确定性阈值、加强结构确认门槛，以后续提案为准）
- 严格限频：限制每日新增开仓次数上限（0/1 为典型候选值，以后续提案为准）

说明：

- `R` 作为参数存在，但本轮不固化数值；后续通过标注与回测校准。

## 8. 审计与回测验证口径（硬边界）

### 8.1 低频纪律的验收

低频纪律的硬含义是：没有机会就不出手；时机出现才进行操作，并且能够拿得住，直到见顶。回测中出现几千次交易属于异常信号，必须被 M4 评估层显式审计，并驱动 M5 提案去压制。

### 8.2 Top200 的正确用途（外部对照基准）

回测验收必须使用“模型推演结果 vs 历史最强 Top200 牛股”的对照逻辑：

- Top200 是外部对照基准（oracle），用于评估模型是否圈定了大多数牛股，且持仓是否接近真正见顶
- Top200 不得作为模型运行的反向框定工具（不得用于限定候选宇宙/入口或引导运行路径）

## 9. 后续迭代方向（不属于 v0 冻结）

后续可增强项：

- 将 `limit_down_proxy_ratio`、`amount_ratio_to_ma5` 纳入触发条件，提升对“放量下杀/跌停潮”的敏感性
- 引入“连续命中”与“熔断式冷却期”机制（仍需保持命中即清仓的硬约束前提下做更稳健的锁定与恢复）
- 引入分市场/分板块的子信号，但市场级 risk-off 仍以全局信号为准
