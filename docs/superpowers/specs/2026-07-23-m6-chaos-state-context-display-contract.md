# 2026-07-23 M6 混沌三层状态上下文最小承接口径

## 0. 目的

本文件只定义 `M6` 对混沌三层状态上下文的最小承接口径：

- 哪些对象需要可见
- 哪些字段只作观测
- 哪些内容禁止被渲染为动作建议
- 当前阶段哪些页面/承接面最适合接住这些对象

本文件不定义：

- 页面实现
- API 路由实现
- 具体 UI 样式
- 具体字段值格式

## 1. 当前阶段目标

当前阶段，`M6` 的职责不是推动混沌动作主线，而是：

- 让三层状态上下文对象可观察
- 让团队拥有共享语境
- 让 `M2/M4` 已冻结的对象与验证结论有统一可见出口

因此，当前阶段的 `M6` 目标应冻结为：

- **最小承接**
- **只读展示**
- **禁止动作化**

## 2. 必须承接的对象

当前阶段，`M6` 至少需要可见的对象为：

### 2.1 stock_state_context

用途：

- 观察单只股票的主语义状态

当前阶段只允许展示为：

- 个股状态读数
- 个股阶段状态
- 个股动态特征

### 2.2 theme_state_context

用途：

- 观察个股所属主题上下文状态
- 观察主题对个股的支持/削弱关系

当前阶段只允许展示为：

- 主题状态读数
- 主题阶段状态
- 主题动态特征

### 2.3 market_state_context

用途：

- 观察全市场背景状态

当前阶段只允许展示为：

- 市场整体背景读数
- 市场阶段状态
- 市场动态特征

### 2.4 state_context_link

用途：

- 解释 L1/L2/L3 的关联关系

当前阶段只允许展示为：

- “该股关联到哪些主题 / 当前市场背景是什么”
- “三层解释顺序与当前可用性”

## 3. 只允许作为观测字段的内容

当前阶段，以下内容只允许作为：

- 观测
- 解释
- 对照

不得渲染成动作建议：

- `core_energy`
- `regime`
- `dynamics`
- `context_validity`
- `coverage_quality`
- `audit_ref`

允许的展示方式：

- 当前状态
- 当前趋势
- 当前背景
- 当前可用性

不允许的展示方式：

- “买入”
- “卖出”
- “加仓”
- “减仓”
- “立即执行”
- 任何带直接交易含义的按钮、标签、文案

## 4. 当前阶段禁止动作化的内容

当前阶段 `M6` 明确禁止：

- 把 `stock_state_context` 直接渲染成入场建议
- 把 `theme_state_context` 直接渲染成板块交易建议
- 把 `market_state_context` 直接渲染成市场择时建议
- 把三层对象的任一字段包装成“推荐/回避”之类动作词

原因：

- 当前阶段混沌模型尚未被正式证明可承担完整动作主线
- `M6` 当前只承担共享观察入口，不承担交易语义生成

## 5. 当前阶段最适合的承接面

### 5.1 MarketIntelligence

当前最适合优先承接：

- `theme_state_context`
- 与其相关的 `market_state_context` 摘要

原因：

- 当前页面已经具有“主题/主线/决策摘要”的承接基础
- 其现有语义与 L2/L3 最接近

### 5.2 OpsCenter

当前适合承接：

- `coverage_quality`
- `context_validity`
- `audit_ref` 的摘要性观测

原因：

- 当前页面已承接系统检查、关键链路状态、异常摘要、运行证据
- 更适合接住“是否 ready / 是否缺失 / 是否可核实”这类信息

### 5.3 当前阶段不建议优先承接的面

当前阶段不建议优先把混沌三层状态对象深接到：

- 完整交易动作工作台
- 任何带人工下单暗示的页面
- 需要过度解释复杂语义的页面

原因：

- 会过早把状态对象动作化
- 会让 `M6` 抢走 `M3` 的语义责任

## 6. 当前阶段最小展示口径

当前阶段建议冻结的展示口径只有三类：

### 6.1 状态卡

用于展示：

- 当前对象状态
- 当前阶段状态
- 当前动态变化方向

### 6.2 上下文关系卡

用于展示：

- 个股 -> 主题 -> 市场 的关系链
- 各层当前是否支持、削弱或只是背景约束

### 6.3 可用性/证据卡

用于展示：

- 当前对象是否 ready
- 当前上下文是否缺失
- 当前证据来源是否完整

## 7. 当前阶段不要求的承接

当前阶段 `M6` 不要求：

- 完整混沌仪表盘
- 复杂历史回放交互
- 多层联动 drill-down
- 动作建议面板

只要求：

- 先接住对象
- 先形成共享观察入口

## 8. 通过条件

本文件当前阶段视为通过，需同时满足：

- 已明确哪些对象必须可见
- 已明确哪些字段只作观测
- 已明确哪些内容禁止动作化
- 已明确当前最适合的承接面

在以上条件满足前，不继续扩展复杂 UI 设计。
