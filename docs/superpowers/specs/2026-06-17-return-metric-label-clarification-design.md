# NeoTrade3 收益指标前端口径澄清（设计定稿）

日期：2026-06-17  
范围：`neotrade3-dashboard` 中 `Overview` 与 `Lowfreq` 页的收益/涨跌文案

## 1. 背景与目标

- 当前前端页面中同时展示了三类不同口径的数字：
  - 虚拟组合累计收益
  - 历史窗口回测收益
  - 市场近 20 日涨跌
- 这些数字目前都以“收益”或“总收益”形式呈现，用户容易把它们误认为同一种指标。
- 尤其 `Lowfreq` 页中的 `20日收益`，当前实际上是市场环境指标 `market_return_20d`，不是策略收益，也不是虚拟组合收益。

目标：
- 在不改变后端字段、不改变计算逻辑的前提下，把前端文案改到足够清楚。
- 明确区分：
  - 当前虚拟组合累计收益
  - 历史窗口回测收益
  - 市场环境近 20 日涨跌
- 保留市场环境指标，但必须明确注明“非策略收益”。

## 2. 设计范围

本次范围：
- `neotrade3-dashboard/src/pages/Overview.jsx`
- `neotrade3-dashboard/src/pages/Lowfreq.jsx`
- 相关前端测试

非目标：
- 不修改后端 API 字段名
- 不变更任何收益计算公式
- 不新增复杂说明弹窗或帮助面板
- 不重构页面布局

## 3. 方案比较

### 3.1 方案 A：就地改名 + 增加短说明

- 保留现有数字与布局。
- 对容易混淆的标题、副标题、说明文案做最小调整。
- 在市场环境指标下明确标注“非策略收益”。

优点：
- 改动最小
- 风险最低
- 最符合当前需求

缺点：
- 仍然依赖用户阅读短说明

### 3.2 方案 B：增加统一“口径说明”区块

- 把三类数字的定义集中写在页面某个说明区域。

优点：
- 信息集中

缺点：
- 用户需要额外对照
- 会增加页面信息密度

### 3.3 方案 C：移除市场环境指标

- 不再显示 `market_return_20d`

优点：
- 完全消除混淆

缺点：
- 丢失一个有价值的市场背景指标
- 不符合当前“保留但写清楚”的确认结论

结论：
- 采用方案 A。

## 4. 实施设计

### 4.1 Overview 页

- `虚拟持仓收益`
  - 保留主标题不变，避免对已有使用习惯造成过大扰动。
  - 将副标题改为更明确的累计口径说明，例如：
    - `当前虚拟组合累计收益`
- `昨日回测（窗口 60 交易日）`
  - 将当前 `总收益` 文案改为：
    - `窗口回测总收益`
  - 保留窗口起止日期显示，用于强调这是区间回测结果，而不是当前账户累计收益。

### 4.2 Lowfreq 页

- 将 `20日收益` 改为：
  - `市场近20日涨跌`
- 在该指标下增加一句短说明：
  - `市场环境指标，非策略收益`
- `投资组合` 区块中当前 `总收益` 改为：
  - `当前组合累计收益`

### 4.3 文案原则

- 文案以“最短可理解”为原则。
- 不使用长段解释。
- 不在同一位置同时混入多个概念。
- 优先通过标题和副标题把口径说清楚，而不是依赖用户自行推断。

## 5. 验证

- 前端测试至少覆盖：
  - `Overview` 页能显示更新后的累计收益/窗口回测文案
  - `Lowfreq` 页能显示 `市场近20日涨跌`
  - `Lowfreq` 页能显示 `市场环境指标，非策略收益`
  - `Lowfreq` 页投资组合区块能显示更新后的累计收益文案
- 运行：
  - `npm run test -- --run`
  - `npm run build`

## 6. 完成标准

- 页面上不再把 `market_return_20d` 简单写成 `20日收益`。
- 用户可以直接从文案区分：
  - 当前组合累计收益
  - 历史窗口回测收益
  - 市场环境近 20 日涨跌
- 前端测试与构建通过。
