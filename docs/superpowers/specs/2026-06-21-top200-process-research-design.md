# Top200 全过程研究设计

## 目标

- 不修改正式交易执行逻辑。
- 基于已存在的两版回测产物，构建 `Top200` 全过程研究底稿。
- 研究重点不是继续增加硬规则，而是回答：
  - 买点是否偏晚、偏追，是否处于较优窗口。
  - 持仓过程中有多少次有效震荡与模型预警。
  - 个股事实见顶与板块、大盘转弱之间的时序关系。
  - 哪些共性适合沉淀为未来的形态化买卖框架。

## 输入口径

- 版本 A：`hold_first_market_tight`
- 版本 B：`hold_first_market_tight + chase block`
- 输入产物：
  - 两版 backtest payload
  - 两版 Top200 attribution JSON
  - `var/db/stock_data.db` 行情数据库

## 输出产物

- 全量轻底稿：
  - 全部 `Top200` 的信号/买入/卖出/见顶轻量研究表
- 重点深挖：
  - 两版中实际买过的 `Top200` 样本逐股档案
- 形态汇总：
  - 基于价格路径、震荡次数、顶点上下文的研究标签汇总
- 研究报告：
  - Markdown 摘要报告

## 核心研究层

### 1. 全量轻底稿

对全部 `Top200` 生成统一字段：

- 年涨幅、主升段起点、事实见顶日期
- 首次信号日期、首次买入日期、首次卖出日期
- 首次信号相对主升段位置
- 首次买入相对主升段位置
- 主升段内 8% 级别震荡次数
- 近似 `A/B/C` 后半段结构
- 原因桶变化（两版对比）

### 2. 已买样本深挖

对两版中实际买过的 `Top200` 样本生成逐股档案：

- 两版模型的实际买卖差异
- 买点位置是否偏早/适中/偏晚
- 持仓内正式卖出信号与研究级预警计数
- `-5%` 正式止损触发情况
- `-6%` 影子观察触发情况与触发后反弹
- 个股事实顶点与市场/板块候选转弱的相对时序

### 3. 研究影子链

在不修改正式执行的前提下，追加研究字段：

- `shadow_stop_6_triggered`
- `shadow_stop_6_first_date`
- `shadow_stop_6_rebound_max_pct`
- `shadow_stop_6_end_return_pct`

该字段仅用于研究，不回写正式卖出行为。

## 结构近似定义

- 主升段起点：沿用现有归因脚本定义，即见顶日前 180 个交易日窗口内最低收盘价。
- 事实顶点：沿用现有归因脚本定义，即年度最高收盘价日期。
- 震荡：主升段内基于收盘价的 8% 级别回撤摆动。
- 波浪近似：
  - 通过价格摆点近似给出 `1/2/3/4/5/A/B/C` 代理节点
  - 若结构不足，则保留部分节点，不强行完整标注
- 买点优劣：
  - 使用买入价在 `主升段起点 -> 事实顶点` 区间内的价格进度作为代理指标
  - 不把该代理指标误当成最终波浪语义结论

## 形态标签

第一轮只给研究标签，不直接绑定执行规则：

- `强趋势单边型`
- `高波动换手型`
- `板块共振型`
- `龙头脱队型`
- `后期加速见顶型`

## 实施边界

- 不改 `lowfreq_engine_v16_advanced.py` 的正式交易行为。
- 优先新增独立研究脚本，避免污染既有归因脚本。
- 先使用现有两版产物完成第一轮研究。
- 只有在研究证明某字段缺失导致结论不稳定时，才回头补埋点。
