# NeoTrade3 股票代码外链跳转（设计定稿）

日期：2026-06-17  
范围：`neotrade3-dashboard` 中所有个股代码展示位置

## 1. 背景与目标

- 当前 `neotrade3-dashboard` 中多个页面会展示代码字段，但这些代码大多还是纯文本。
- 用户要求把前端页面中的股票代码做成可点击链接。
- 点击后不做站内弹窗，不做嵌入式 K 线，而是直接新开外部窗口，展示大陆公开网站上的个股详情/K 线页面。
- 已确认默认外部站点为：
  - `同花顺`

目标：
- 在 `neotrade3-dashboard` 中，把所有 A 股个股代码统一渲染为可点击外链。
- 点击后直接新开同花顺对应个股详情页。
- 概念代码、板块代码、主题代码保持纯文本，不误跳转。

## 2. 范围

本次范围：
- `neotrade3-dashboard` 中所有直接展示个股代码的位置
- 新增统一前端组件或工具函数，用于：
  - 识别 A 股个股代码
  - 生成同花顺目标 URL
  - 统一渲染链接样式
- 对已有页面中的个股代码展示节点进行替换
- 补充相关前端测试

非目标：
- 不实现站内弹窗
- 不做 iframe 嵌入外部 K 线页
- 不支持概念代码、板块代码、主题代码的外链跳转
- 不扩展到港股、美股、ETF、指数
- 不引入站点切换器

## 3. 方案比较

### 3.1 方案 A：统一组件，逐位置替换

- 新增统一组件，例如 `StockCodeLink`
- 由组件内部完成：
  - 个股代码识别
  - URL 构造
  - `window.open()` 跳转
- 页面中所有个股代码位置改为使用该组件

优点：
- 语义清晰
- 可控且易测
- 不会误伤非个股代码
- 后续若要切换站点，改动集中

缺点：
- 需要逐个页面替换已有渲染节点

### 3.2 方案 B：文本扫描式自动替换

- 在页面层或通用包装器中扫描文本，把像代码的字符串自动替换成链接

优点：
- 表面上覆盖更快

缺点：
- 极易把概念代码、板块代码、主题代码误判为个股代码
- 难以控制边界
- 不适合当前项目对精确性的要求

### 3.3 方案 C：只覆盖表格区域

- 只在表格中的代码列改成链接
- 卡片、摘要区、详情区维持纯文本

优点：
- 实现更快

缺点：
- 不满足“前端页面上所有股票代码”的目标

结论：
- 采用方案 A。

## 4. 设计

### 4.1 识别规则

- 只给 A 股个股代码加链接。
- 首批按当前前端实际数据形态识别 6 位纯数字代码，且限定在常见 A 股前缀：
  - `000`
  - `001`
  - `002`
  - `003`
  - `300`
  - `301`
  - `600`
  - `601`
  - `603`
  - `605`
  - `688`
 - 对以下代码保持纯文本：
  - 概念代码，如 `886042.TI`
  - 板块代码，如 `BK001`
  - 主题/其他非个股代码

### 4.2 跳转规则

- 默认站点固定为 `同花顺`。
- 点击个股代码后，前端使用安全的新窗口打开外部详情页：
  - `target="_blank"`
  - `rel="noreferrer noopener"`
- 若当前展示值不是受支持的个股代码，则渲染为普通文本，不生成链接。

### 4.3 URL 生成规则

- 为避免不同页面各自拼接 URL，统一抽到一个工具函数中。
- 工具函数输入：
  - 原始代码字符串
- 工具函数输出：
  - 若是受支持 A 股个股代码，返回同花顺 URL
  - 否则返回 `null`
- 站点域名与路径以当前公开可访问规则为准，统一集中维护，不分散写死在页面里。

### 4.4 组件边界

- 推荐新增：
  - `src/components/StockCodeLink.jsx`
- 组件职责：
  - 接收 `code`
  - 判断是否支持跳转
  - 可跳转时渲染链接
  - 不可跳转时渲染纯文本
- 组件不负责：
  - 解析概念/板块代码
  - 决定业务字段含义
  - 站内路由跳转

### 4.5 页面替换范围

- 逐页替换 `neotrade3-dashboard` 中所有当前显示个股代码的位置。
- 对明显不是个股代码的字段不替换，例如：
  - `concept_code`
  - `BKxxx`
  - 主题/板块编号

## 5. 验证

- 前端测试至少覆盖：
  - 个股代码会渲染为可点击链接
  - 非个股代码保持纯文本
  - 链接使用预期外站 URL
- 运行：
  - `npm run test -- --run`
  - `npm run build`

## 6. 完成标准

- `neotrade3-dashboard` 中所有展示 A 股个股代码的位置都可点击。
- 点击后直接新开同花顺对应外部详情页。
- 概念代码、板块代码、主题代码不被误处理。
- 前端测试和构建通过。
