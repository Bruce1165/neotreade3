# 早窗龙头系统性退出保护 v2 设计

## 目标

在 `system_exit_grace v1` 已被长窗验证否证后，重新收窄保护资格。

v2 的目标不是“给所有有利润垫的龙头多一次机会”，而是只保护这类样本：

- 早窗或前置布局进入；
- 在主升中段被系统性确认打断；
- 当前利润仍保留了大部分峰值成果；
- 更像“主升过程中的误伤”，而不是“高位回吐后只剩残余盈利”。

## v1 失败结论

长窗 A/B 结果：

- 基线 `hold_first_market_tight + chase_block`
  - 收益 `57.66%`
  - 最大回撤 `13.78%`
- `+ system_exit_grace v1`
  - 收益 `37.11%`
  - 最大回撤 `17.30%`

Top200 变化：

- `bought_count: 3 -> 2`
- `held_to_top_count: 2 -> 0`

说明：

- v1 虽然减少了一部分止损并拉长了持仓，但整体是明确负优化；
- 失败根因不是“系统性确认降级”这个方向完全错误，而是保护资格太宽。

## 证据对照

### 目标保护样本

#### 1. `603119 浙江荣泰`

- 首次买入：`早窗`
- 基线卖出后到事实顶剩余上涨：`237.56%`
- 市场确认相对顶点：`-14` 天
- 核心特征：
  - 早窗进入
  - 市场确认来得非常早
  - 卖出后空间巨大

#### 2. `688519 南亚新材`

- 首次买入：`前置布局`
- 基线卖出后到事实顶剩余上涨：`182.38%`
- 板块确认相对顶点：`+15` 天
- 核心特征：
  - 极早布局
  - 个股卖出过早，但最终板块确认并不早
  - 更像板块早期波动误伤

#### 3. `301345 涛涛车业`

- 首次买入：`早窗`
- 首段卖出后剩余上涨：`26.63%`
- 市场确认相对顶点：`-6` 天
- 核心特征：
  - 早窗进入
  - 已获取大额收益
  - 仍有剩余上涨，但不是无限空间

### 对照结论

v1 误保护样本的共同点：

- 并不是“早窗 + 主升误伤”的典型；
- 更多是利润已经明显回吐，只剩低个位数或低双位数利润；
- 即使历史峰值收益不低，也不代表当前仍适合保护。

因此 v2 的核心不是继续盯 `peak_return_pct`，而是增加“形态型”和“利润保真型”限制。

## v2 资格语义

### 1. 形态资格

必须满足以下任一：

- 首次买入为 `早窗`
- 首次买入为 `前置布局`

不再保护：

- 首次买入已经落在 `中窗` 或 `晚窗`

### 2. 利润保真资格

必须同时满足：

- 当前收益仍为正
- `profit_keep_ratio = current_return_pct / peak_return_pct`
  - 不低于某个保真阈值

建议起点：

- `SYSTEM_EXIT_GRACE_MIN_PROFIT_KEEP_RATIO = 0.50`

也就是：

- 若历史曾盈利很多，但当前只剩下不到一半峰值利润，不保护

### 3. 范围资格

只保护首次系统性确认来得足够早的样本。

建议用系统性确认与事实顶点的相对时差做代理：

- 若已有研究基准样本证据，优先保护：
  - `top_market_pass_offset <= -5`
  - 或 `top_sector_pass_offset >= 0` 且个股卖出明显早于事实顶

在线实现时没有事实顶点，因此运行时代理需要换成更简单条件：

- 只保护“首次确认时持仓仍处于主升保持态”的样本
- 可由以下代理组合近似：
  - 当前收益不低于 `10%`
  - 当前收益占峰值收益比例足够高
  - 最近未触发个股级弱化证据

## market / sector 拆分

v2 不再让 `market_top_confirmed` 和 `sector_top_confirmed` 共享完全同一套保护资格。

### 1. `market_top_confirmed`

保护条件更严格：

- 必须 `早窗/前置布局`
- 必须 `profit_keep_ratio >= 0.50`
- 必须当前收益不低于 `10%`

### 2. `sector_top_confirmed`

保护条件更窄：

- 只保护 `前置布局` 或非常明确的 `早窗`
- 必须 `profit_keep_ratio >= 0.60`
- 必须当前收益不低于 `15%`

理由：

- v1 里错误保护的 sector 样本较多；
- 板块确认误伤更像“极早期波动误杀”，所以只该保护最早、最强的个股。

## v2 最小实现

只在 v1 资格函数基础上新增 3 个判断：

- `buy_progress_label in {"早窗", "前置布局"}`
- `current_return_pct >= min_current_profit_pct`
- `profit_keep_ratio >= min_profit_keep_ratio`

并将 `scope == "market"` 与 `scope == "sector"` 的阈值拆开。

建议参数：

- `SYSTEM_EXIT_GRACE_MARKET_MIN_CURRENT_PROFIT_PCT = 10.0`
- `SYSTEM_EXIT_GRACE_MARKET_MIN_PROFIT_KEEP_RATIO = 0.50`
- `SYSTEM_EXIT_GRACE_SECTOR_MIN_CURRENT_PROFIT_PCT = 15.0`
- `SYSTEM_EXIT_GRACE_SECTOR_MIN_PROFIT_KEEP_RATIO = 0.60`

## 本轮非目标

本轮仍不解决：

- 再介入逻辑
- 仓位保护/容量优先级
- 部分减仓
- 独立的个股见顶规则

## 验证重点

若进入 v2 实现，第一轮只看三件事：

- 收益是否不再像 v1 那样明显劣化
- `held_to_top_count` 是否至少不继续下降
- 被保护样本是否从“低利润残余票”收窄到真正早窗强票
