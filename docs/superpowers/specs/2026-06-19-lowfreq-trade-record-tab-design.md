# 低频交易页签改为交易记录与白页修复设计

## 背景

低频交易页面当前存在一个页签 `portfolio`，文案为“持仓监控”。该页签实际承载的数据不止当前持仓，还包含：

- 组合概览
- 当前持仓
- 已完成交易记录
- 人工干预记录

用户反馈点击该页签后出现白页面。同时，从现有实现可见，`PortfolioPanel` 存在多处对可能缺失字段直接调用 `.toFixed()` 的代码路径，任何字段缺失都可能在 React 渲染期抛错并导致白页。

## 目标

- 将页签文案从“持仓监控”改为“交易记录”，使入口语义与实际承载内容一致。
- 保留现有数据源与主要展示结构，不新增接口，不扩展新交互。
- 修复因 `portfolio` 子字段缺失导致的渲染期白页。
- 为该页签补充点击回归测试，覆盖“进入页签不白页”和“缺字段时降级显示”。

## 非目标

- 不重做为时间线/流水视图。
- 不新增交易过程聚合接口。
- 不改动其他低频交易页签的数据获取逻辑。
- 不调整人工干预与交易记录的业务含义。

## 现状证据

`PortfolioPanel` 当前会无保护读取如下字段：

- `portfolio.total_value`
- `portfolio.cash`
- `portfolio.total_return_pct`
- `portfolio.initial_capital`
- `pos.market_value`
- `pos.unrealized_pnl_pct`

这些字段一旦缺失，会在调用 `.toFixed()` 时抛出异常，导致整个页签白页。

同时，页签内部已经展示“交易记录”和“人工干预记录”，说明“交易记录”比“持仓监控”更能准确概括该页签承载的信息。

## 方案选择

### 方案 A：文案与信息架构最小调整，同时补空值防护

实施内容：

- 页签文案：`持仓监控 -> 交易记录`
- 保留 `portfolio` 作为内部状态键与数据来源，避免扩大改动面
- 页面结构按“交易过程理解顺序”展示：
  - 组合概览
  - 当前持仓
  - 已完成交易
  - 人工干预记录
- 对所有数值展示增加安全格式化与缺值兜底
- 补充点击页签的前端回归测试

优点：

- 风险最小
- 不需要新增后端接口
- 能同时解决命名不准与白页问题

缺点：

- 仍然不是完整的时间线交易流水视图

### 方案 B：重做为完整交易流水视图

不采用。原因是改动面过大，与本次定点修复范围不匹配。

## 推荐方案

采用方案 A。

原因：

- 与用户诉求直接一致
- 与现有数据结构匹配
- 能在最小改动下消除当前白页故障
- 验证边界清晰，可快速回归

## 设计细节

### 1. 导航文案

低频交易页签数组中，将 `portfolio` 对应的 `label` 从“持仓监控”改为“交易记录”。

说明：

- 仅修改用户可见文案
- 不修改 `activeTab === 'portfolio'` 的内部状态键，避免联动修改过多条件分支

### 2. 页面结构

保留现有四个内容区块，但文案上以“交易记录”页签承载完整过程视角：

- 组合概览：说明当前组合状态
- 当前持仓：说明交易过程当前仍在运行的仓位
- 已完成交易：显示已平仓记录
- 人工干预记录：显示人工介入轨迹

本次不改区块业务含义，只校正文案与入口语义。

### 3. 白页修复

对 `PortfolioPanel` 内所有数值展示引入统一的安全格式化策略：

- 金额字段：
  - 有限数值时显示格式化金额
  - 缺值或非数值时显示 `--`
- 百分比字段：
  - 有限数值时显示百分比
  - 缺值或非数值时显示 `--`
- 普通文本字段：
  - 空字符串或缺值时显示 `--`

重点覆盖：

- 概览卡片
- 策略信息栏
- 持仓明细表
- 交易记录表

### 4. 交互与降级

当 `portfolio` 存在但部分字段缺失时：

- 页面继续渲染
- 对应单元格显示 `--`
- 不再抛出渲染期异常

当 `portfolio` 整体不存在时：

- 继续沿用现有“暂无持仓数据”降级面板

## 测试计划

新增或调整前端定点测试：

1. 点击“交易记录”页签后，页面继续渲染，不出现白页
2. `portfolio` 仅提供最小字段时，页签仍可打开
3. 缺失 `initial_capital`、`market_value`、`unrealized_pnl_pct` 等字段时显示 `--`
4. 既有“已完成交易”“人工干预记录”仍按原语义展示

## 风险与控制

风险：

- 文案变化可能影响既有测试选择器
- 数值兜底若处理不一致，可能让部分字段出现展示口径不统一

控制：

- 测试中优先按按钮文案与可见标题验证
- 统一使用同一组安全格式化 helper，避免散落判断

## 验收标准

- 低频交易页签显示“交易记录”
- 点击该页签不再白页
- 缺字段数据下页面显示 `--` 而不是崩溃
- 现有持仓、已完成交易、人工干预记录内容仍可见
